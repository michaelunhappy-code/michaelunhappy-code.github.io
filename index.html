1. 巧克力广告调查

小明最近看起了电视广告。一则广告上说：“我们的 XX 巧克力，$100\%$ 的孩子都爱吃！” 于是小明调查起了 XX 巧克力，发现他们只对一个孩子做过测试，他爱吃......（$1 / 1 = 100\%$）

于是，小明就想：照他们这个逻辑，如果说 “$50\%$ 的孩子都爱喝”，那么他们有可能只对两个孩子做过测试；同样地，如果说 “$20\%$ 的孩子都爱喝”，那么他们有可能只对五个孩子做过测试。

那么，如果说，$P\%$ 的孩子都爱喝（$P$ 是一个正整数），那么至少需要对几个孩子做测试才有可能呢？（这里，我们认为 $P\%$ 是一个精确的数，并不是通过四舍五入得到的数）

💡 题解分析

本题的核心是求一个最简分数的隐式分母。题目给定了一个百分数 $P\%$，它可以表示为分数 $\frac{P}{100}$。假设总共调查了 $X$ 个孩子，其中有 $Y$ 个孩子爱吃，那么一定满足 $\frac{Y}{X} = \frac{P}{100}$。

为了求出“最少”需要测试的孩子数量，实际上就是要求解等式右边分数的**最简形式**下的分母。我们将分子分母同时除以它们的最大公约数（$\gcd$）即可：

$$X_{min} = \frac{100}{\gcd(P, 100)}$$

代码中通过辗转相除法实现求最大公约数 `gcd(a, b)` 函数，然后直接输出 `100 / gcd(P, 100)` 即为答案。

输入格式

输入一行一个正整数 $P$。

输出格式

输出一个正整数 $X$，表示至少需要对 $X$ 个孩子做测试。

#include <bits/stdc++.h>
using namespace std;
int gcd(int a,int b){
    return b? gcd(b,a%b):a;
}
int main(){
    // freopen("ad.in","r",stdin);
    // freopen("ad.out","w",stdout);
    int x;
    cin>>x;
    cout<<100/gcd(x,100)<<endl;
    return 0;
}

也就是说，对于 $(2^k)*(2^k)$ 的板，他要穿 $k$ 次线，第 $i$ 次穿线要穿 $2^i$ 根线。

在穿线的过程当中，小明想知道相邻的两个点之间有没有线相隔。你能帮忙回答这个问题吗？

💡 题解分析

这是一道找规律的题目。初始板子大小为 $2^k \times 2^k$。每次执行穿线（$0$ 操作）后，相当于把空间的分割线密度加倍。

通过观察可以发现：设当前已经执行了 $cnt$ 次 $0$ 操作，那么相邻两根线之间的距离刚好是 $2^{k-cnt}$。如果我们要判断相邻的两个点之间是否有线相隔，我们只需找出这两个点在这条维度上较小的那个坐标值（设为 $minn$）。

因为题目保证两点相邻，即绝对值之差为 1，只要较小的那个坐标 $minn$ 能够被当前线间距 $2^{k-cnt}$ 整除（即 `minn % (1 << (k-cnt)) == 0`），那么这两个点之间就必定存在一条分割线。通过 `min()` 判断两点所在的相对方位，并用位运算（或 `pow2` 函数）计算步长，就能以 $O(1)$ 的时间复杂度快速回答每个询问。

输入格式

第一行，两个整数 $k,m$，表示 $N=2^k$，一共有 $m$ 个操作。
接下来 $m$ 行，有两种操作：
$0$ 表示穿了一次线。
$1$ $x_1$ $y_1$ $x_2$ $y_2$ 表示询问 $(x_1,y_1)$ 与 $(x_2,y_2)$ 之间有无线相隔。
保证 $|x_1-x_2|+|y_1-y_2|=1$，且 $0$ 操作的数量不超过 $k$ 次。

输出格式

对每个 $1$ 操作，输出一行一个 $0$ 或 $1$，$1$ 表示有线相隔，$0$ 表示无线相隔。

#include <bits/stdc++.h>
using namespace std;
int k,m;
int cnt;
int pow2(int x){
    return (1<<x);
}
int main(){
    freopen("bend.in","r",stdin);
    freopen("bend.out","w",stdout);
    cin>>k>>m;
    while(m--){
        int id;
        cin>>id;
        if(id==0){
            cnt++;
        }
        else{
            int x1,y1,x2,y2;
            cin>>x1>>y1>>x2>>y2;
            if(x1==x2){
                int minn=min(y1,y2);
                if(minn%(pow2(k-cnt))==0){
                    cout<<1<<endl;
                }
                else cout<<0<<endl;
            }
            else{
                int minn=min(x1,x2);
                if(minn%(pow2(k-cnt))==0){
                    cout<<1<<endl;
                }
                else cout<<0<<endl;
            }
        }
    }
    return 0;
}

3. 哆啦A梦的楼梯

小明梦见了哆啦A梦。

哆啦A梦在楼梯的顶端（第 $N$ 级），而小明在楼梯的底部（第 $0$ 级）。小明每次最多可以跨 $K$ 级。但是，有几级楼梯是坏的（保证第 $N$ 级楼梯是好的）。小明一旦踏上去，就会坠入万丈深渊。

你能帮他算算，有多少种跨楼梯的方式，最终可以让他与来到楼梯的顶层和哆啦A梦相聚吗?

💡 题解分析

这是一道经典的**动态规划（DP）**爬楼梯问题的变形。常规的爬楼梯是每次走 $1$ 或 $2$ 步，而本题允许走 $1$ 到 $K$ 步，且有若干不可达的“坏楼梯”。

定义状态 $dp[i]$ 表示到达第 $i$ 级楼梯的方案数。状态转移方程为：

$$dp[i] = \sum_{j=\max(0, i-K)}^{i-1} dp[j]$$

在处理前，将所有损坏的楼梯位置标记为特殊的不可达状态（例如 `INF`）。在进行状态转移时，如果当前考察的 $dp[i]$ 是 `INF`，说明此处不可站立，直接 `continue` 跳过。同样，累加前面楼梯方案数时，也要避开那些标记为 `INF` 的楼梯点。初始条件 $dp[0] = 1$，且每次累加都需要对 $1000000007$ 取模，防止整型溢出。

输入格式

第一行三个整数 $N$，$K$，$Z$。其中 $Z$ 表示坏的楼梯的个数（可能为 $0$）。
接下来一行 $Z$ 个正整数，表示哪些楼梯级是坏的，不能走（$Z$ 个正整数升序读入，且两两互不相同）。

输出格式

输出一行一个整数，表示方案数除以 $1000000007$ 的余数。

#include <bits/stdc++.h>
using namespace std;
const int N=1e5+5;
const int MOD=1e9+7;
const int INF=0x3f3f3f3f;
typedef long long LL;
LL my_max(LL a,LL b){
    return a>b? a:b;
}
LL dp[N];
LL err[N];
LL n,k,z;
int main(){
    freopen("feeling.in","r",stdin);
    freopen("feeling.out","w",stdout);
    cin>>n>>k>>z;
    for(LL i=1;i<=z;i++){
        cin>>err[i];
        dp[err[i]]=INF;
    }
    dp[0]=1;
    for(LL i=1;i<=n;i++){
        if(dp[i]==INF){
            continue;
        }
        for(LL j=my_max(0,i-k);j<i;j++){
            if(dp[j]!=INF){
                dp[i]+=dp[j];
                dp[i]%=MOD;
            }
        }
    }
    cout<<dp[n]<<endl;
    return 0;
}

4. 二元一次方程组求解

给定 $N$ 个二元一次方程。

第 $i$ 个二元一次方程为：$A_i * x + B_i * y = C_i$

如果这 $N$ 个方程形成的方程组无解，输出 “No solution!”

如果有唯一解，输出两行（其中 $a,b$ 为常数，保留三位小数）：

x=a
y=b

如果不止一组解，输出“OK”。

💡 题解分析

本题考察对多元方程组解的判断，本质上是求解一次直线系统的交点。由于方程以浮点数形式输入和比对，涉及精度问题，必须使用一个极小值 `eps`（通常设为 `1e-8`）来判断两数是否相等。

  1. 初筛： 如果读入的方程出现 $A=0, B=0$ 但 $C \neq 0$，立刻判定为不合法，直接输出 No solution!。
  2. 克莱姆法则求交点： 使用两层循环暴力枚举所有有效的直线组合。对于任意两条直线，计算行列式 $d = A_1B_2 - A_2B_1$。
    • 如果 $d \neq 0$（即 fabs(d) > eps），说明两条直线不平行，必有唯一交点 $(x, y)$。此时立刻求出 $(x, y)$，并将这组解代入验证剩余所有的方程。如果全都满足，输出此唯一解；只要有一个不满足，说明有多条不共点的直线，整体无解输出 No solution!。
    • 如果 $d = 0$，说明这两条直线平行或重合。如果常数项不成比例（平行不相交），说明出现矛盾方程组，直接判定 No solution! 并结束。
  3. 多解判断： 若以上所有的枚举都未产生唯一交点（意味着所有存在的合法直线都互相重合），则表示方程组有无数组解，最后输出 OK。

输入格式

第一行一个整数 $N$。
接下来 $N$ 行，每行三个整数（可能为 $0$ 或负数），第 $i$ 行的整数为 $A_i,B_i,C_i$。

输出格式

详见问题描述。

/*
伪代码：
定义方程结构体
    输入（double）
    如果方程是0x+0y=z 直接No Solution!
    否则加入方程组
    枚举每两个方程
        如果不是矛盾的系数
            解出来
            对每个方程验证
                如果不满足 -> No Solution
            如果所有的全部满足 -> 输出 x,y
    
    最终 -> 输出“OK”
    所有的运算符(double)
    "!=0" -> ">eps" "==0" -> "<eps"
    输入使用double,fabs()
*/
#include <bits/stdc++.h>
using namespace std;
const double eps=1e-8;
struct P{
    double a,b,c;
};
int main(){
    freopen("equation.in","r",stdin);
    freopen("equation.out","w",stdout);
    int n;
    cin>>n;
    vector<P> v;
    for(int i=0;i<n;i++){
        double a,b,c;
        cin>>a>>b>>c;
        if(fabs(a)<eps&&fabs(b)<eps){
            if(fabs(c)>eps){
                cout<<"No solution!"<<endl;
                return 0;
            }
        }
        else{
            v.push_back({a,b,c});
        }
    }
    for(int i=0;i<v.size();i++){
        for(int j=i+1;j<v.size();j++){
            double d=v[i].a*v[j].b-v[j].a*v[i].b;
            if(fabs(d)>eps){
                double x=(v[i].c*v[j].b-v[j].c*v[i].b)/d;
                double y=(v[i].a*v[j].c-v[j].a*v[i].c)/d;
                for(auto t:v){
                    if(fabs(t.a*x+t.b*y-t.c)>eps){
                        cout<<"No solution!";
                        return 0;
                    }
                }
                cout<<fixed<<setprecision(3)<<"x="<<x<<endl;
                cout<<fixed<<setprecision(3)<<"y="<<y<<endl;
                return 0;
            }
            else{
                if(fabs(v[i].a*v[j].c-v[j].a*v[i].c)>eps||fabs(v[i].b*v[j].c-v[j].b*v[i].c)>eps){
                    cout<<"No solution!";
                    return 0;
                }
            }
        }
    }
    cout<<"OK"<<endl;
    return 0;
}
